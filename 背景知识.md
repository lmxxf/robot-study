# 机器人学习 - 背景知识

> 学习过程中的 Q&A 记录，帮助理解 VLA、具身智能、模型训练等核心概念。

## Q1：现在主流的 10万~20万人形机器人，用什么水准的模型？

### 架构：大脑 + 小脑

现在主流是双层架构，不是一个模型打天下：

| 层 | 功能 | 模型类型 | 参数量级 | 算力需求 |
|---|------|---------|---------|---------|
| **大脑** | 理解指令、任务规划 | VLA/VLM（视觉语言动作模型） | 数亿~数百亿 | 可以云端 |
| **小脑** | 实时运动控制、平衡 | RL 策略网络 | 数百万~数千万 | 必须本地，≤200ms |

国家标准要求家用机器人算力 ≥100 TOPS，工业机器人算力 ≥500 TOPS。

### 具体案例

- **宇树 G1/R1**（2.1万~3万起）：开源 UnifoLM-VLA-0，本地算力 ≥100 TOPS
- **Figure 03**：自研 Helix VLA 模型，端到端路线
- **特斯拉 Optimus**：FSD 视觉架构 + xAI Grok 做语言理解
- **智元机器人**：具身大模型 GO-1，2025 年出货 5168 台全球第一

### 和桌面级方案对比

| | 商用人形机器人 | DGX Spark + SO-101 |
|---|---|---|
| 大脑模型 | 私有 VLA，数亿~数十亿参数 | SmolVLA 450M，同一量级 |
| 小脑模型 | RL 策略，几百万参数 | ACT，同一思路 |
| 训练数据 | 企业级采集+仿真，亿级 | 社区数据+自己录 50 集 |

**结论：模型本身够用，差距在数据量和硬件形态。**

### 行业趋势

模型在往小走：机器人本体算力有限，云端有延迟。大模型负责"想"（可以慢），小模型负责"动"（必须快）。

## Q2：为什么叫 VLA？认知基础不再是语言模型了吗？

### 认知基础还是语言模型

VLA 不是取代语言模型，是语言模型长出了眼睛和手：

```
LLM  (Language Model)         只会说话
 ↓
VLM  (Vision-Language Model)  会看+会说    ← GPT-4V, Gemini
 ↓
VLA  (Vision-Language-Action)  会看+会说+会动  ← 机器人专用
```

VLA 的骨架就是一个预训练好的 VLM，然后在上面接了一个动作输出头。

### SmolVLA 的结构

```
SmolVLM-2 (预训练的视觉语言模型，骨架)
    ↓
+ Vision Encoder (看摄像头画面)
    ↓
+ Language Input (听指令："把杯子放到盘子上")
    ↓
+ Action Expert (flow-matching 动作头，输出关节角度序列)
```

不是三个独立模型拼起来的，是一个语言模型，视觉和动作是后长出来的器官。

### 为什么不叫 LAM

因为视觉是刚需。纯语言模型控制机器人 = 闭着眼干活。之前的方案是视觉/语言/控制三个模块分开，靠工程师写接口对接，脆弱且每个场景都要调。VLA 把三件事塞进一次前向传播里，端到端。

## Q3：VLA 也是"原生多模态"吗？

### 不全是，得看具体实现

| 模型 | 视觉怎么接的 | 算不算原生 |
|------|------------|-----------|
| **RT-2** (Google) | 底座是 PaLM-E 55B，视觉从头联训 | 接近原生 |
| **π0** (Physical Intelligence) | 基于预训练 VLM，视觉和动作联合微调 | 半原生 |
| **SmolVLA** | 骨架是 SmolVLM-2，动作头是后接的 | 视觉原生，动作后挂 |
| **大多数开源 VLA** | 拿现成 VLM 冻住，外面套 action decoder | 纯后挂 |

现状是一个光谱：

```
纯后挂                                          纯原生
  |————————————————|————————————————|————————————|
  冻住VLM+外挂头    VLM微调+action头   从头联训全部模态
  (便宜,效果一般)                      (贵,效果最好)
```

### 各模态的"原生"程度

- **V（视觉）**：基本都是原生或深度融合，因为继承自 VLM
- **L（语言）**：原生，Transformer 骨架
- **A（动作）**：几乎都是后挂的。没有模型在预训练阶段就输出关节角度——动作数据太稀缺

**现在的 VLA 是一个有眼睛有大脑、但手是后装的生物。** 等哪天从预训练就把动作模态塞进去（视频预测→物理世界模型→动作输出全打通），才是真·原生 VLA。

## Q4：VLA 的动作后训练不会灾难性遗忘吗？

### 问题有多严重

π0 团队实测：动作头随机初始化的梯度回传到 VLM 骨架后，直接把语言理解能力搞崩了——机器人不再能正确理解指令。

根本原因：VLM 预训练数据 = 互联网级别图文（十亿级），动作微调数据 = 机器人遥操作轨迹（百万级），分布差异太大。

### 四种主流解法

#### 1. 冻住 VLM，只训动作头（SmolVLA）

```
VLM 骨架 (350M) ——— 完全冻结 ❄️
       ↓
  动作专家 (100M) ——— 只训这部分 🔥
```

最简单，完全不碰 VLM 权重，不可能遗忘。SmolVLA 在 LIBERO 上照样打赢了 π0。

#### 2. 把动作编码成语言 token（OpenVLA）

```
关节角度 [0.12, -0.34, 0.56, ...]
        ↓ 离散化
动作 token: <a_12> <a_neg34> <a_56> ...
```

不加动作头，用 VLM 自己的输出层生成"动作词"。对 VLM 来说输出动作和输出文字是同一件事，分布冲突小。

#### 3. 知识隔离 / Knowledge Insulation（π0.5）

```
VLM 骨架 ←── 只接收离散 token 的梯度 ✅（温和）
    ↓
动作专家 ←── flow-matching 连续动作 🔥
    ↑
  梯度截断！动作专家的梯度不回传到 VLM ❌🚫
```

目前最精巧的方案。VLM 通过离散 token 学"该做什么"（语义级），动作专家学"具体怎么动"（控制级），两路梯度互不干扰。还能同时混入 VL 数据做 co-training。

#### 4. LoRA 微调 + 数据对齐（VLM2VLA）

先把动作翻译成自然语言描述（"手臂向左移动 3cm"），然后用 LoRA 微调。数据格式和预训练一致，LoRA 只动 0.1% 权重，遗忘极小。

### 风险-能力光谱

```
遗忘风险：低 ────────────────────────── 高
          |          |          |          |
        冻住VLM    LoRA     知识隔离    全参数微调
       (SmolVLA) (VLM2VLA)  (π0.5)     (π0 初版,翻车了)
          |          |          |          |
表达能力：弱 ────────────────────────── 强
```

## Q5：大模型预训练完成后，初始 SFT 阶段怎么做 replay？

### 核心做法：往 SFT 数据里混预训练数据

```
SFT 训练的一个 batch：
├── 90-99% 指令微调数据（问答对、对话、指令跟随）
└──  1-10% 预训练原始数据（网页文本、代码、论文...）
          ↑ 这就是 replay
```

InstructGPT（ChatGPT 前身）论文里就写了这个做法——在 SFT 和 RLHF 阶段都混入预训练数据。

### 混多少

| 比例 | 效果 |
|------|------|
| **0%（不混）** | 会遗忘，通用能力下降明显 |
| **1%** | 已经有显著改善 |
| **~10%** | 甜区，大多数团队用这个量级 |
| **GeoGalactica** | 8:1:1（领域数据:论文:代码） |
| **太多** | SFT 效率下降，指令跟随学不好 |

### 大部分"遗忘"是伪遗忘

2025 年重要发现：SFT 阶段大部分"遗忘"其实是 **spurious forgetting（伪遗忘）**。

```
预训练 base 模型输出：
  "法国的首都是巴黎。巴黎位于塞纳河畔..."（续写模式）

SFT 之后输出：
  "法国的首都是巴黎。"（对话模式，简洁回答）

Benchmark 测试：得分下降了！
```

知识没丢，是回答风格变了导致 benchmark 对不上。不是知识擦除，是对齐漂移。

### 真正会丢的是长尾知识

模型预训练时只见过几次的冷门知识（如小语种语法规则），SFT 阶段几轮梯度更新就能覆盖掉。高频知识（英语语法、常识推理）反而很难丢。

### 主流防遗忘工具箱

```
① Data Replay（混预训练数据）      ← 最基础，人人都用
② LoRA / QLoRA（只动 0.1% 权重）  ← 动得少，忘得少
③ KL 散度正则化（和原模型对比）    ← RLHF 阶段常用
④ 学习率 rewarm + re-decay        ← 避免初期大梯度冲击
⑤ Sa-SFT（自己生成 replay 数据）  ← 没有预训练数据时，让模型自己"回忆"
```

### 工业现实

**拥有预训练数据 = 拥有防遗忘的万能钥匙。** 这就是为什么 OpenAI/Google/Meta 对预训练数据集严格保密——不是怕你看到内容，是怕你拿到数据后可以对他们的模型做无损微调。DeepSeek 开源了权重但没开源预训练数据，也是这个道理。

## Q7：人类对"运动神经"理解多少？比对 LLM 理解得多吗？

### 结论：解剖清楚，原理模糊（和 LLM 正好反过来）

| | 运动神经 | LLM |
|---|---|---|
| 研究历史 | **200+ 年**（1870 年发现运动皮层） | **~10 年** |
| 硬件结构 | **清楚** | 清楚（Transformer） |
| "哪块管什么" | **清楚** | 不清楚（attention head 功能不明） |
| 信号怎么传 | **清楚** | 清楚（前向传播） |
| 为什么能学会 | **部分清楚** | 部分清楚（梯度下降，但涌现不明） |
| 内部表征是什么 | **不清楚** | 不清楚 |

### 搞清楚的部分：层级架构

```
运动皮层 (Motor Cortex)     <- "想动"，规划意图
    |
小脑 (Cerebellum)           <- 实时纠错，"想做的"vs"实际做的"偏差
    |
基底神经节 (Basal Ganglia)  <- 选择动作方案，抑制不该做的
    |
脊髓 (Spinal Cord)          <- 执行，自己也能计算（不只是中继）
    |
运动神经元 -> 肌肉           <- Sherrington 说的"最终公共通路"
```

每一层损坏会怎样，临床验证了无数次：
- 小脑损伤 -> 共济失调（手抖、走不直）
- 基底神经节损伤 -> 帕金森（动不了）或亨廷顿（停不下来）
- 脊髓损伤 -> 瘫痪
- 运动皮层损伤 -> 对侧偏瘫

### CPG（中枢模式发生器）理解得尤其透彻

脊髓里的 CPG 回路不需要大脑指令就能产生节律性运动——走路、游泳、呼吸。把猫的脑切断放跑步机上，它还能走。数学模型（耦合振荡器）已经非常成熟，直接用到了机器人上（波士顿动力四足、仿生蛇、仿生鱼）。

### 没搞清楚的部分：从意图到动作的编码

你想"拿杯子"，运动皮层几百万个神经元怎么协调出精确的手臂轨迹？知道有"方向调谐"和"群体编码"，但不知道"概念"怎么变成"轨迹"——和 LLM 不知道"理解"怎么涌现是同一个困境。

### 与 VLA 的对应关系

```
人类运动系统              VLA 机器人
-------------------------------------------------
运动皮层（意图+规划）  <->  VLM 骨架（理解指令+视觉）
小脑（实时纠错）       <->  还没有对应物！
基底神经节（动作选择） <->  attention 机制
CPG（节律运动）        <->  ACT/Diffusion Policy
脊髓（执行+反射）      <->  PID 控制器
```

注意：VLA 目前没有"小脑"——没有实时"我想做 X，实际做了 Y，差多少"的在线纠错回路。

**人类对运动神经的理解是"解剖清楚、原理模糊"，对 LLM 的理解是"原理清楚、行为模糊"。正好反过来。**

## Q8：Transformer 做运动神经太慢，VLA 需要更快的运动神经吗？

### 需要。Transformer 是大脑，不是运动神经

| | 人类小脑 | Transformer (VLA) |
|---|---|---|
| 反馈频率 | **1000 Hz** | **10-50 Hz** |
| 延迟 | ~10ms | 50-200ms |
| 能耗 | ~20W（整个大脑） | 200-400W（一张 GPU） |

VLA 不直接驱动电机。中间有传统 PID 控制器在 1000Hz 做伺服。VLA 每 50ms 扔出一段动作块（action chunk），PID 负责平滑跟踪。ACT 的 "Action Chunking" 就是因为 Transformer 太慢，只能批发不能零售。

### 致命问题：没有小脑

VLA 说"接下来 0.5 秒手往左走"，执行到一半杯子被挪走了，PID 还在傻傻跟踪旧轨迹。要等下一次 VLA 推理才能反应。

### 正在出现的架构：快慢双系统

```
慢系统（大脑皮层）                快系统（小脑/脊髓）
VLA / Transformer                轻量网络 (MLP/CNN/SNN)
~10-50 Hz                        ~200-1000 Hz
理解指令、看画面、规划            触觉反馈、力控、防撞、姿态修正
几亿参数                          几百万参数
GPU                               CPU / 神经形态芯片
```

三个具体方案：
1. **Reactive Diffusion Policy**（上海交大+清华，RSS 2025）：慢网络规划 action chunk，快网络（可学习的阻抗控制器）根据触觉/力反馈实时修正
2. **DuoCore-FS**（异步快慢 VLA）：VLM 慢路径 ~5Hz，动作快路径 ~100+Hz，各跑各的不互相等
3. **脉冲神经网络 SNN**（Intel Loihi 2）：神经形态芯片，毫瓦能耗，微秒延迟，硬件模拟生物神经元

## Q9：机器人蹦蹦跳跳很流畅，但意图是傻子——运动神经不是挺成熟了吗？

### 关键区分：locomotion（腿）vs manipulation（手）

| | 全身运动 (locomotion) | 手部操作 (manipulation) |
|---|---|---|
| 例子 | 走、跑、翻跟头、跳 | 拿杯子、拧螺丝、叠衣服 |
| 自由度 | 少（腿就那几个关节） | 多（手指 20+ 自由度） |
| 成熟度 | **很成熟** | **很不成熟** |
| 核心技术 | RL + MPC，已量产 | VLA + 触觉反馈，还在实验室 |

### 为什么 locomotion 先成熟了

```
locomotion:
  - 周期性运动 -> CPG/RL 天然适配
  - 自由度少 -> 搜索空间小
  - 不需要视觉（可以盲走）-> 纯本体感知就够
  - 成功标准简单 -> "没摔倒" = reward
  - 仿真->真机 gap 小 -> 地面接触模型够用

manipulation:
  - 非周期性 -> 每个物体都不一样
  - 自由度爆炸 -> 灵巧手 20+ 关节
  - 强依赖视觉+触觉 -> 多模态融合
  - 成功标准模糊 -> "拿稳了"怎么定义？
  - 仿真->真机 gap 巨大 -> 柔性物体、摩擦力仿真不准
```

### 实际案例

- **宇树 H1**：3.3 m/s 世界纪录，RL sim-to-real 一把过
- **波士顿动力 Atlas**：MPC + RL 混合控制，翻跟头跑酷
- 技术栈标准化：仿真 RL -> sim-to-real -> 量产

### 现在的人形机器人身体状态

```
腿：RL + MPC，成熟，200-1000Hz 闭环      <- "蹦蹦跳跳"
手：VLA + PID，半成品，10-50Hz           <- "笨手笨脚"
脑：VLM/VLA，婴儿期                      <- "意图是傻子"
```

**下半身是大学生，上半身是幼儿园，脑子是婴儿。**

### locomotion vs manipulation 技术栈对比

| | locomotion（腿） | manipulation（手） |
|---|---|---|
| 控制器 | RL 策略（小 MLP）+ MPC | VLA（Transformer）+ PID |
| 频率 | 200-1000 Hz | 10-50 Hz |
| 模型大小 | 几百万参数 | 几亿参数 |
| 训练方式 | 仿真 RL，几小时搞定 | 模仿学习，需要人类示教数据 |
| 感知 | IMU + 关节编码器（不用眼睛） | 摄像头 + 触觉（必须用眼睛） |
| 成熟度 | 量产就绪 | 实验室阶段 |

这也解释了为什么宇树先卖四足狗（纯 locomotion）最成功，人形机器人商业落地主要是"走来走去搬箱子"（locomotion + 简单抓取），而不是"叠衣服做饭"（精细 manipulation）。

## Q10：RL、MLP、MPC、PID 都是什么？

> 机器人控制技术栈里反复出现这四个缩写，逐个解释。

### MLP — Multi-Layer Perceptron（多层感知机，就是全连接网络）

最古老最简单的神经网络。每一层的每个神经元和下一层的每个神经元全部连接。

```
输入层        隐藏层        输出层
[关节角度]    [256个神经元]   [关节力矩]
[角速度  ] -> [256个神经元] -> [12个输出]
[IMU数据 ]    [256个神经元]
```

为什么机器人腿部控制用 MLP 而不用 Transformer：
- 输入就几十个数字（关节角度、角速度、IMU），不是序列，不需要 attention
- 推理极快：几十微秒，能跑 1000Hz
- 几百万参数就够用
- Transformer 处理序列数据是杀鸡用牛刀

**类比：MLP 是计算器，Transformer 是电脑。算 1+1 不需要开电脑。**

### RL — Reinforcement Learning（强化学习）

不是一种网络结构，是一种**训练方法**。和 SFT（监督微调）是平级概念：

```
监督学习 (SFT)：给正确答案，让模型学 -> "老师教你写字"
强化学习 (RL)：给奖惩信号，让模型自己摸索 -> "扔水里学游泳"
```

机器人腿部控制的 RL 训练过程：

```
1. 在 MuJoCo 仿真里放一个机器人
2. 机器人随机乱动（MLP 网络随机初始化）
3. 没摔倒 -> +1 分（reward）
4. 摔了 -> -10 分
5. 走得快 -> +5 分
6. 重复几百万次
7. MLP 的权重通过梯度更新，越来越会走路
```

**关键区别**：
- VLA 用**模仿学习**（看人怎么做，学人的动作）——需要人类示教数据
- 腿部控制用 **RL**（自己在仿真里试错）——不需要人类数据，reward 函数就够

为什么腿用 RL 而手用模仿学习：
- 走路的 reward 好定义："没摔倒 + 走得快" -> 简单的数字
- 抓东西的 reward 不好定义："优雅地拿起杯子" -> 怎么量化"优雅"？

### MPC — Model Predictive Control（模型预测控制）

MPC 不是神经网络，是**传统控制理论**，纯数学优化，不需要训练。

核心思想：**每一步都往前预测几步，选最优的动作。**

```
当前状态：机器人正在往右倒

MPC 的大脑（每毫秒都在算）：
  "如果我左脚往左迈 5cm -> 预测 0.2 秒后：还是倒 -> 不行"
  "如果我左脚往左迈 10cm -> 预测 0.2 秒后：稳了 -> 选这个！"
  "如果我左脚往左迈 15cm -> 预测 0.2 秒后：过头了 -> 不行"

输出：左脚往左迈 10cm
```

需要什么：
- 一个**物理模型**（机器人的质量、关节、惯性矩阵——工程师手写）
- 一个**优化器**（在线求解，每步都重新算）

优缺点：

```
优点：
  - 不需要训练数据
  - 可解释（每一步都是数学最优解）
  - 安全性好（可以硬编码约束："关节角度不能超过 X 度"）

缺点：
  - 需要精确的物理模型（模型不准就废了）
  - 复杂地形/未知环境扛不住（没见过的情况算不出来）
  - 计算量大（预测步数越多越慢）
```

**波士顿动力的 Spot/Atlas 就是 MPC 起家的。** 后来发现纯 MPC 在复杂地形上不行，才加了 RL。现在是 MPC + RL 混合：RL 负责"在仿真里学会应对各种意外"，MPC 负责"在线实时优化保证安全"。

### PID — Proportional-Integral-Derivative（比例-积分-微分控制器）

最简单的控制算法，三行公式，没有神经网络，没有训练，没有学习。1920 年代发明的，至今仍是工业控制的基石。

核心思想：**看偏差，修正偏差。**

```
目标：关节角度 = 90 度
当前：关节角度 = 85 度
偏差：e = 90 - 85 = 5 度

PID 输出力矩 = Kp * e + Ki * 积分(e) + Kd * 微分(e)
             = "偏了多少" + "偏了多久" + "偏差变化快不快"

P（比例）：偏 5 度 -> 使点劲
I（积分）：一直偏 -> 加大力度（消除稳态误差）
D（微分）：偏差在缩小 -> 别使太猛（防止超调振荡）
```

调参就是调 Kp、Ki、Kd 三个数字，没有什么"训练"。

PID 在机器人里的位置：**最底层，离电机最近。**

```
VLA："接下来关节 1 走到 90 度"
         |
PID：1000Hz 不停算偏差，调力矩，让关节跟上目标
         |
电机：转
```

**PID 不懂任何语义，它只知道"目标值"和"当前值"的差。** 就像空调的温控——设 26 度，高了制冷，低了制热，不需要知道你为什么要 26 度。

### 四者的关系（从上到下）

```
层级          技术          类比
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
决策层        VLA/Transformer    大脑：看+听+想，决定"拿那个杯子"
                 |
规划层        MPC               小脑：规划轨迹，"手从 A 沿曲线到 B"
                 |
策略层        RL + MLP           脊髓：学会的运动模式，"走路时腿怎么配合"
                 |
执行层        PID               运动神经元：跟踪目标角度，闭环伺服
                 |
硬件          电机+减速器        肌肉
```

不是每一层都必须有。实际组合：
- **腿部控制**：RL(MLP) + MPC + PID（三层，不需要 VLA，不需要看）
- **手部控制**：VLA + PID（两层，跳过了 MPC 和 RL，直接端到端）
- **未来理想**：VLA + MPC/RL + Reactive Policy + PID（四层全上）
